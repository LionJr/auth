# Auth

REST API для аутентификации пользователей по email.

## Что делает

Сервис реализует вход по одноразовому коду (OTP), отправляемому на почту через SMTP. После проверки кода выдаёт пару JWT-токенов (access и refresh). При обновлении токенов отслеживается IP-адрес клиента — при его смене отправляется email-уведомление.

**Эндпоинты:**

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/sign-in` | Запрос кода на email |
| POST | `/api/auth/check-code` | Проверка кода и выдача токенов |
| POST | `/api/auth/refresh` | Обновление access/refresh токенов |
| GET  | `/ping` | Проверка работоспособности |

Swagger UI: `http://localhost:8080/swagger/index.html`

## Требования

- Go 1.25+
- PostgreSQL
- Redis
- SMTP-сервер

## Запуск

1. Применить миграцию к базе данных:

```bash
psql -h <host> -U postgres -d auth -f schema/000001_init.up.sql
```

2. Настроить параметры подключения в `configs/local.yml` (PostgreSQL, Redis, SMTP, ключи JWT).

3. Запустить сервер:

```bash
go run ./cmd -e local
```

Флаг `-e` задаёт окружение и выбирает файл конфигурации `configs/<environment>.yml` (по умолчанию `local`).

Сервер слушает порт из конфига (по умолчанию `8080`). Для корректного определения IP клиента передавайте заголовок `X-Forwarded-For`.
